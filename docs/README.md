# Envoy documentation

Start with the [dogfooding guide](dogfooding.md). It explains how to introduce Envoy into a repository, coordinate parallel issue work, interpret safety findings, and handle common failure scenarios.

The documentation has three levels of authority:

- [Dogfooding guide](dogfooding.md): task-oriented instructions for operators and coding agents.
- [README](../README.md): installation, the shortest successful workflow, and command overview.
- [Product specification](../spec.md): normative behavior, persisted models, safety rules, and roadmap boundaries. If a guide and the specification disagree, the specification wins.

[plan.md](../plan.md) tracks delivery slices. A planned feature is not available merely because it appears in that file. Future commands and behaviors are labeled explicitly in the dogfooding guide.
